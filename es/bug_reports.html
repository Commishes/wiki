Reportar bugs

Si estás aquí, probablemente encontraste un error en nuestro software y deseas informarlo. Y estamos muy agradecidos de que lo estés haciendo. Para ayudarnos a ayudarte mejor, sigue atentamente estas instrucciones; un buen informe de error nos ayudará MUCHO a hacer de Commishes un mejor lugar para todos.

Al momento de escribir este artículo, estamos usando Github issues; por lo tanto, deberá registrar una cuenta en Github para que podamos realizar un seguimiento adecuado del informe de error.

Paso 1: Identifica la aplicación que causa el problema

Commishes se compone de varios servicios más pequeños, estas aplicaciones colaboran para brindar el servicio. Mantenemos repositorios separados para las diferentes aplicaciones, por lo tanto, te agradeceríamos que informaras los problemas en la aplicación correcta para clasificar rápidamente su error.

  • Para cuestiones relativas a YCH.Commishes.com, CRM.commishes.com y del correo electrónico; envía un informe de errore aquí: https://github.com/Csardelacal/YCH-Issues/issues
  • Para problemas con account.commishes.com, por favor envía el informe aquí: https://github.com/Csardelacal/PHPAuthServer/issues
  • Para problemas con ping.commishes.com cy relacionados a las notificaciones, envía el informe aquí: https://github.com/Csardelacal/ping/issues

Paso 2: comprueba si tu problema está presente

Esto es realmente importante para nosotros. Unos segundos de tu tiempo para ver si tu problema aparece en la lista nos ahorra muchos gastos administrativos y hace maravillas con nuestra eficiencia. Al hacer clic en uno de los enlaces anteriores, verá una lista de problemas. Github también proporciona una búsqueda de informes de errores.

Si tu problema aparece en la lista, agrega un comentario al problema si tienes información adicional o haz clic en el ícono "Me gusta" para informarnos que el problema también está presente para ti. Si tu problema ya está presente, entonces ya está. Alguien revisará el problema y lo abordará lo antes posible.

Paso 2.1: Informar un error o presentar una solicitud de función

Una vez que hayas iniciado sesión en Github, encontrarás que tienes la opción de hacer clic en un botón verde con la etiqueta "Nuevo problema". ¡Clickea en el botón! Para archivar correctamente el error, recuerda incluir todo lo siguiente:

  • Información aproximada del sistema. ¿Estás en una computadora de escritorio o móvil? ¿Qué navegador y versión estás usando? Por ejemplo: Windows 7/Internet Explorer 11
  • Incluye la URL en la que aparece el error. Si tu URL contiene tokens o información de sesión, elimínalo primero.
  • Proporciona capturas de pantalla si es necesario. Elimina cualquier información de identificación personal. Los problemas son públicos.
  • Incluye lo siguiente en tu informe: ¿Qué estabas tratando de hacer? ¿Qué pasó? ¿Qué se suponía que iba a pasar?

Puedes utilizar este formulario para redactar su informe: (Recuerda que toda la información deberá estar en Inglés)

 Platform: [desktop|table|mobile] Operating system: [Android, iOS, Windows, MacOS, Linux] Browser: [IE, Firefox, Chrome, Safari, Other] URL: What were you trying to do: What was supposed to happen: What happened:

 

Step 3: Envía el informe, revisa tu correo electrónico

Simplemente haz clic en "Enviar". Nosotros nos encargaremos de todo lo demás. Tu error será clasificado (se le dará prioridad) y recibirás actualizaciones cada vez que algo cambie. Github te actualizará automáticamente cada vez que haya alguna actualización en tu informe de error. Deberás estar atento a tus correos electrónicos; es posible que necesitemos que hagas un seguimiento de tu informe.

Step 4: Disfruta de nuestra gratitud

Nuevamente, los informes de errores son la mejor manera de ayudarnos a mejorar y mejorar. No podemos solucionar problemas si no sabemos que están ahí. ¡Realmente nos estás ayudando! ¡Muchas gracias!
